# Regression Testing

## Definisi

Regression Testing merupakan metode Gray Box Testing yang dilakukan setelah adanya perubahan atau perbaikan sistem untuk memastikan fitur yang sebelumnya berjalan dengan baik tidak mengalami kerusakan akibat perubahan tersebut.

## Tujuan

- Memastikan fungsi lama tetap berjalan normal.
- Mengidentifikasi bug baru setelah perbaikan.
- Menjaga stabilitas sistem.
- Memastikan integrasi sistem tetap berjalan dengan baik.

## Objek Pengujian

Fitur Login, Register, dan Home pada sistem EMart.

## Penjelasan Pengujian

Pengujian dilakukan setelah adanya perubahan pada file JavaScript yang mengatur validasi Login dan Register. Tester melakukan pengujian ulang terhadap seluruh fitur utama sistem.

## Tabel Pengujian

| No | Fitur | Sebelum Perbaikan | Setelah Perbaikan |
|----|---------|---------|---------|
| 1 | Login | Berjalan | Berjalan |
| 2 | Register | Berjalan | Berjalan |
| 3 | Home | Berjalan | Berjalan |

## Hasil Pengujian

Tidak ditemukan kesalahan baru setelah dilakukan perubahan pada sistem.

## Kesimpulan

Seluruh fitur tetap berjalan normal setelah proses perbaikan sehingga sistem dinyatakan stabil.
